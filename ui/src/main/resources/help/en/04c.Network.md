# Network

## NAT

NAT means [Network Address Translation](https://en.wikipedia.org/wiki/Network_address_translation). The system that allows many devices behind your router to reach the Internet using the single public IP address of your router. The problem with NAT is that incoming connections don't work without some help.

### External IP and Port

The external IP and port should be displayed here, in the case it was detected properly.

### Use UPNP

[Universal Plug and Play (UPNP)](https://en.wikipedia.org/wiki/Universal_Plug_and_Play) is a software that runs on your router. If enabled, Xeres will try to use it to do 2 things:

- get the external IP
- open an incoming port

Both are desired to get good connections to your friends. It's recommended to enable it when on your own private network (at home), and disabled otherwise. On Windows, if set to *Private* (the default), it will be enabled automatically if your network uses a private profile (
see [how to set a private profile on Windows](https://search.brave.com/search?q=how+to+set+a+network+profile+on+windows)). On other platforms, *Private* will default to *off*.

While UPNP got a bad reputation because of some early bugs, if your router is recent enough, you can safely enable it there.

#### Fallback to external DNS

If checked, and if UPNP fails to get the external IP, Xeres will try to use a random public DNS from [OpenDNS](https://en.wikipedia.org/wiki/OpenDNS) or [Akamai](https://en.wikipedia.org/wiki/Akamai_Technologies). These services will know you want to find out your public IP, but this is done by many other applications
as well.

## LAN

LAN means [Local Area Network](https://en.wikipedia.org/wiki/Local_area_network). Your own network behind your router.

### Internal IP and Port

The internal IP and port are displayed here.

### Use LAN Discovery

LAN Discovery is a system to find other users on the local network. It periodically sends a UDP broadcast packet to find out if other Xeres clients are on the LAN. It implements the same protocol as found in the [udp-discovery-cpp](https://github.com/truvorskameikin/udp-discovery-cpp) project.

The following information is broadcast:

- a Xeres application identifier
- our IP address and port
- our profile fingerprint
- the location identifier
- the profile name (user's nickname)

The data is used to update a friend's IP on the LAN to be able to connect to them if it changed (for example a DHCP lease that expired). If enabled in the Notifications settings, it can also display that someone is available on the LAN.

Since the information might be sensitive, it's recommended to only enable it when on your own private network (at home), and disabled otherwise. On Windows, if set to *Private* (the default), it will be enabled automatically if your network uses a private profile (
see [how to set a private profile on Windows](https://search.brave.com/search?q=how+to+set+a+network+profile+on+windows)). On other platforms, *Private* will default to *off*. There might be other scenarios where enabling it might be useful (for example a LAN party).

## DHT

The [Distributed Hash Table](https://en.wikipedia.org/wiki/Distributed_hash_table) is a system that helps two peers find each other when their IP address has changed or is unknown. It is useful especially in a dynamic environment (roaming, dynamic IP address, ...)

Xeres uses the [Mainline DHT](https://en.wikipedia.org/wiki/Mainline_DHT) which is the DHT used by [BitTorrent](https://en.wikipedia.org/wiki/BitTorrent). It will periodically send and receive UDP packets with other random peers to store a hash of your location ID. Other friends will request that ID and will eventually
find you. Note that your IP address itself is never stored in the DHT.

### Enable DHT

Enable or disable the DHT.

When to enable the DHT:

- you have a IP address that frequently changes
- you use a laptop and roam/travel a lot
- you have many friends that also have those habits

When not to enable the DHT:

- you have a fixed/stable IP address and a working UPNP
- you have a very limited bandwidth
- you want to stay discrete. DHT will send packets to many peers

By default, it is disabled.

## Hidden Services

This setting is only to be able to connect to Retroshare clients that are using hidden services ([Tor](https://en.wikipedia.org/wiki/Tor) or [I2P](https://en.wikipedia.org/wiki/I2P)). Xeres itself does not implement any kind of hidden service so this won't anonymize anything.

If you want to connect to Tor or I2P users, respectively configure the Tor/I2P socks proxy to your running Tor/I2P installation.
