# 启动参数

当手动运行Xeres时，您可以使用以下命令选项。这些仅用于高级用法，通常不需要使用。

- `--no-gui`：无界面启动。可用于在无头模式下运行Xeres。配合`--remote-connect`参数使用另一个实例来连接它。
- `--iconified`：启动时最小化到系统托盘。这对自动启动很有用。
- `--data-dir=<路径>`：指定数据目录。这是Xeres存储所有用户文件的位置。如果要运行多个实例，每个实例都需要有不同的数据目录。
- `--control-address=<主机>`：指定用于接收远程访问的绑定地址（默认为仅本地主机）。
- `--control-port=<端口>`：指定用于远程访问的控制端口。这是UI将连接的端口。如果要运行多个实例，每个实例都需要有不同的控制端口，但Xeres会自动尝试寻找空闲端口（从1066开始），因此这个参数很少需要使用。
- `--no-control-password`：不对控制地址进行密码保护。密码在首次运行时自动生成，并可在设置中查看。密码可以更改或禁用。
- `--server-address=<主机>`：指定要绑定的本地地址（如果未指定，则绑定到所有接口）。
- `--server-port=<端口>`：指定用于接收传入连接的本地端口。默认情况下，Xeres会随机选择一个端口并在同一实例中永久使用。
- `--fast-shutdown`：忽略正常的关闭程序。这主要用于需要快速运行/关闭Xeres实例的测试场景。正常使用时不需要。
- `--server-only`：仅接受传入连接，不建立传出连接。主要用于聊天服务器。
- `--remote-connect:<主机>[:<端口>]`：作为UI客户端启动并连接到指定节点。您也可以在局域网上跨机器执行此操作。请注意，该连接未加密。如要克服这一限制，请使用SSH隧道。
- `--remote-password=<密码>`：远程连接时使用的密码
- `--version`：打印软件版本
- `--help`：打印帮助信息
